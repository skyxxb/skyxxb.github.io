Web

Easy Injection

题目给了一个url和hint：为什么查询语句大写

访问url后表单可以输入username和password，输入任意user和pwd，返回:

Hello,’用户名’

当username = admin，任意password时返回

Wrong password.

猜测需要获得pwd或者admin的身份

python模板注入

其中username存在模板注入，通过访问config全局变量得到SECRET_KEY的值

config[SECRET_KEY]+cookie

这里涉及到另一个知识点，flask中使用SECRET_KEY加密某些数据得到cookie

通过任意注册用户得到的cookie解出明文，是输入的username，那么可以加密’admin’得到admin的cookie

eyJ1c2VybmFtZSI6ImFkbWluIn0.XNZT0w.ZZEA-69I9r2ejYxq1DX6O46dzaU

获得admin身份成功登陆后得到下一步提示

继续利用cookie访问该url，?id=1时返回

可以看出id存在sql注入，经过测试，waf会检测空格、union、select、flag关键字

空格使用+进行替换

unicode 大小写 绕过

Object?

题目给出game.php源码

通过getLog()访问$log_ 并输出

反序列化绕过

使用unserialize()，__destruct()，存在反序列化漏洞

但是在init()中进行输入过滤

第一个通过序列化数组绕过

第二个通过php反序列化的bug绕过，在对象长度前添加一个+号，即o:14->o:+14

参考：http://www.phpbug.cn/archives/32.html

payload:

a:2:{s:4:”name”;s:4:”test”;i:0;O:7:”BlogLog”:1:{s:4:”log_”;s:5:”/flag”;}}

把o:7->o:+7，然后编码[urlencode(‘+’)->%2B]

a%3A2%3A%7Bs%3A4%3A”name”%3Bs%3A4%3A”test”%3Bi%3A0%3BO%3A%2B7%3A”BlogLog”%3A1%3A%7Bs%3A4%3A”log_”%3Bs%3A5%3A”%2Fflag”%3B%7D%7D

访问

http://10.112.100.47:8233/game.php?data=a%3A2%3A%7Bs%3A4%3A%22name%22%3Bs%3A4%3A%22test%22%3Bi%3A0%3BO%3A%2B7%3A%22BlogLog%22%3A1%3A%7Bs%3A4%3A%22log_%22%3Bs%3A5%3A%22%2Fflag%22%3B%7D%7D

得到flag

注意

序列化的结果需要url编码

保存flag的文件需要爆破，常见:/flag、/flag.txt、/flag.php等

JustUpload

题目需要上传文件，经过测试，只能上传图片类型文件（白名单），对文件名后缀采用黑名单限制，对上传的文件内容和长度进行限制（len<=15）

后缀绕过

php->php4

文件类型绕过

在Content-Type、文件头进行判断

Content-Type：image/jpeg

上传图片头标识位：

jpg —> FFD8FF
png —> 89504E47 0D0A1A0A

内容绕过

上传的文件内容中不允许出现php关键字，使用php短标签绕过，同时许多关键字可以被特殊字符替代

本来使用最短一句话就行

但是内容长度<=15，由于图片头已经占了3位，php标签占6位，剩下写入命令的长度只剩6位，所以不能上传一句话

可以尝试通过反引号直接命令执行

注意

得到flag的命令使用的是

ls /

[APK]Open Sesame!

反编译apk，分析java层代码，对输入的key进行check，找到libnative-check.so

jnihelloword

使用rsaEncode()对key进行加密，与encrypt字符串进行比较。猜测encrypt是加密后的字符串，key是原始字符串

rsaEncode

核心部分见图

对输入的key的每个字符做rsa加密，每次循环e次

题目中给出了e、n，解密需要得到d，由于加密时间复杂度低同时明文长度短，可以直接猜解flag

动态调试

在解密之前需要得到encrypt字符串内容，需要动态调试

且需要绕过反调试

Hello_World

附件是一个Hello_world.docx，打开发现没啥有用信息

通过binwalk分析得到Flag.xml，里面有很多经过不同编码的字符串

只要你能老老实实学会这些Hello world的写法，就一定能找到你想要的东西。

第一段编码采用Brainfuck

解密得到

https://www.zhihu.com/question/63387608

访问某乎关于如何有格调地输出「Hello, World!」？

第一个回答跟颜文字相关，正好题目待解码文件有颜文字，直接解码（颜文字是js的aacode 编码）

解得flag
